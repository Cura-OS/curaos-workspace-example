# Shared: Local CI Merge Gate (GH auto-CI OFF)

Canonical statement of the local-CI-first merge gate. Rule: `ai/rules/curaos_local_ci_first_rule.md`; runbook: `ai/curaos/docs/ci-local.md`. Both [prompts](../milestone-orchestration-prompt.md) and the [one-task prompt](../one-task-execution-prompt.md) point here instead of restating it.

- GitHub auto-CI is OFF: org Actions billing is exhausted and the 7 CI workflows are `workflow_dispatch`-only. There is NO green GitHub check behind a PR; the LOCAL gate IS the merge gate.
- The canonical gate set = every BLOCKING gate in `curaos/ci-gates.yaml` (the single source of truth for CI gates). Run via `just ci` (or `just ci-service <name>` / `just ci-changed`) from the `curaos/` root, equivalently `bash curaos/scripts/ci-local.sh` without `just`. Do NOT hand-pick or freeze a command list; the blocking gates are whatever the config currently marks `blocking: true` for the touched scope (today: `install`, `lint`, `secret-scan-staged`, `commitlint`, `ci-gates-sync`, `typecheck`, `knip`, `syncpack`, `publint`, `forward-only-migrations`, `depcruise`, `aggregate-ci`, plus the Tier C/D/E blocking gates per scope). The config drives the runner; read it, do not duplicate it.
- `node scripts/check-ci-gates-sync.js` (run from `curaos/`) must be green: it proves `ci-gates.yaml` has not drifted from the dispatch-only GH `tier-*.yml` workflows, so a green local run genuinely simulates CI (local gate == GH workflow definition). On drift, fix the config, never the consumers.
- `gh pr checks <n> --watch` has no required check to wait on and returns immediately; it only meaningfully waits when a workflow was MANUALLY dispatched. Trigger GH CI manually ONLY when genuinely needed (`env -u GITHUB_TOKEN gh workflow run <wf>.yml --repo <repo> --ref <branch>`), never as a routine per-PR gate. Do NOT treat an empty/auto-passing `gh pr checks` as the gate; the verbatim local re-run + green sync-check IS the gate.
- A `check:<owner/repo>:<workflow>@<ref>` requires-entry is satisfied by a manually dispatched green run; absent a dispatch, the verbatim local re-run of the equivalent `ci-gates.yaml` blocking gate is the standing evidence; do not block solely on a missing auto run.

Evidence shape (who pastes what): the worker pastes verbatim stdout per [one-task prompt](../one-task-execution-prompt.md) section 8.1 (last 15 lines + real exit code per blocking gate; the pasted counts are the claim of record). The orchestrator re-runs the same gates at the same path per milestone prompt section 7.1 (over-claim re-run gate, +-2 test tolerance).
