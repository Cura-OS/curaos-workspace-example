# admin_app — Agent Context

## Quick facts
- **Framework:** React 18 + Next.js 14 App Router
- **Recipe:** `ui.react-next` (ADR-0153)
- **Status:** Migrating from Flutter; no production Flutter code to preserve
- **Key consumers:** Platform operators, tenant admins

## Architecture notes
- Shell: root layout renders `@curaos/ui` `<AdminShell>` (sidebar + topbar).
- Protected routes gated by `@curaos/auth-sdk` `withAuth` HOC or RSC-level session check.
- Dynamic admin pages (App Builder output) embed via RSC server components or iframe sandbox.
- All data fetching uses React Query hooks generated by `@curaos/api-client`.
- Turborepo `build` and `dev` scripts; lint via ESLint + Biome; type-check via `tsc --noEmit`.

## Key files (post-migration)
- `app/layout.tsx` — root shell layout
- `app/(admin)/tenants/` — tenancy management routes
- `app/(admin)/users/` — user/role routes
- `app/(admin)/audit/` — audit log viewer
- `app/(admin)/plugins/` — plugin registry
- `lib/auth.ts` — server-side session helper (wraps `@curaos/auth-sdk`)
- `components/` — page-level components not yet in ui_kit

## Agent rules
- No PHI/PII fields in admin_app — tenancy and user metadata only.
- Auth flows delegate to CuraOS Auth portal (ADR-0120); no local credential storage.
- Plugin registry pages read-only in v1; write operations require workflow approval step.
- Use `@curaos/ui` components before writing custom UI.
- Run `turbo run build lint test e2e` before marking done.
