# Auto-Decision Log

> Append-only quick log of decisions an agent **auto-applied** under [[curaos-recommendation-auto-apply-rule]] (user directive 2026-05-29: *"If you ever have a recommended action never escalate the question to me and go with the recommendation"*). Each row is a decision the agent took **without interrupting the user** because a clear recommendation existed. This file is the durable, scannable record so the user can review every silent choice later - and revise, revert, or promote one to a binding ADR if they disagree.

**This is NOT a full ADR.** It is a lightweight ledger. When a logged decision needs to become binding (cross-cutting, shapes architecture, others must follow it), promote it to a numbered ADR under `ai/curaos/docs/adr/` and link the ADR back to its log row.

## How agents append here (binding)

Whenever an agent (orchestrator, worker, or any lane) auto-applies a recommended option instead of escalating, it MUST add one row to the table below **in the same unit of work that applies the decision** - before the decision binds downstream. Skipping the log entry is a rule violation: a silent decision with no log entry is invisible to the user and defeats the point.

Row fields:
- **Date** - `YYYY-MM-DD` the decision was applied.
- **Context** - where it arose (issue `repo#N`, milestone tag, ADR question, or task), one line.
- **Options** - the candidate paths considered (so the user can see what was on the table).
- **Chosen** - the option auto-applied.
- **Why** - the one-line reason it was the clear recommendation (research finding, low blast radius, convention match).
- **Reversible-how** - how to undo/switch later if the user wants a different answer (the escape hatch - feature flag, config, follow-up issue, ADR override).
- **Ref** - link to where the decision binds (issue comment, commit SHA, lane registry, PR).

When the user reviews a row and wants a different answer, they say so; the agent treats that as a normal user decision, reverses via the Reversible-how path, and adds a follow-up row noting the override.

## Escalate instead of logging when

The agent does NOT auto-apply + log (it escalates via `AskUserQuestion` per §3.6) when:
- **No recommendation exists** - a genuine trade-off where analysis prefers no path.
- **Irreversible / destructive / T3-class** - the destructive-op confirmation gate survives any recommendation (`AGENTS.md` §11).
- **Scope expansion** - work not in project docs or approved this session: propose first.
