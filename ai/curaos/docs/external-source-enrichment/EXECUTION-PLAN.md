## Scope counted

- 42 new MISSING issues (XSRC-M-01..42, this closeout) - deduped vs the 178

## Governing invariants
- **Partition key = `owner_path` (git working tree).** Same owner_path => serialize; distinct => parallel.
- **Concurrency cap = min(16, cores-2) = 16 lanes** (18-core host).
- **Generator-first barrier.** While ANY `tools/codegen` emitter, `@curaos/*-sdk`, or `@curaos/contracts` (.tsp) lane is in-flight, every downstream service/app that consumes that mold WAITS - it would inherit the defect (2-3x hot-fix cost). Foundations gate service waves.
- **Person + management dual surface from one contract** via ui-app-emit, every capability. The person journey is the spine; management/compliance attaches to it. No source capability dropped (preserved or version-filed-forward).
- **Demo/runtime data is DB-backed** via gen:service-seed - no runtime API mocks (mocks only for unit tests + CI e2e harness).
