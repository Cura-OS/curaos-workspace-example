Helm semantics cited:
- Helm v3 chart hooks: `pre-install` runs before normal resources; hooks are sorted by weight, kind, then name; lowest weight loads first; Helm waits for hook readiness; Job and Pod hooks must successfully complete; hook failure fails the release and blocks the Helm client. Source: https://v3.helm.sh/docs/v3/topics/charts_hooks/
- Helm v3 hook deletion policies: `before-hook-creation` deletes the previous resource before a new hook launches; `hook-succeeded` deletes after success; `hook-failed` deletes the resource if the hook failed. Source: https://v3.helm.sh/docs/v3/topics/charts_hooks/
- Helm v3 install timeout: `--timeout` waits for individual Kubernetes operations, including Jobs for hooks, default `5m0s`. Source: https://v3.helm.sh/docs/v3/helm/helm_install/
