### P2

1. Explicit empty `changeValues` map can still be emitted by direct publisher callers.

Evidence:
- Publisher omits only when `input.changeValues === undefined`: `curaos/backend/services/identity-service/src/identity-core/audit/audit-publisher.service.ts:198-200`.
- Schema uses optional partial-record shape, so the type is a partial map rather than a non-empty object: `curaos/backend/services/identity-service/src/identity-core/audit/audit-event.schema.ts:144-149`.

Impact:
- Not a current PHI leak and not a slice-2 live-path regression: `actors.service.ts` does not populate `changeValues` at all. But a future direct caller could pass `{}` and get a serialized `changeValues: {}` key, so the strongest form of "no key unless meaningful values exist" is not enforced by the publisher.

Recommendation:
- Non-blocking. If desired in #115, normalize empty maps to `undefined` before spread or add a small publisher guard that only emits when at least one key has a non-empty array.
