# Codex grill — M8-S1 PRs curaos#88 + curaos-ai-workspace#134

## Verdict: BLOCK

## P0 findings (block merge)

### P0-1 — Bundle is pinned to amd64 even though air-gap contract is multi-arch

Evidence:
- `ops/zarf/zarf.yaml:32` sets `metadata.architecture: amd64`.
- `ai/rules/curaos_airgap_rule.md:34` says Zarf supports "Multi-arch image bundling (amd64 + arm64)".
- `ai/rules/curaos_airgap_rule.md:53` shows canonical package definition with `architecture: multi  # amd64 + arm64`.
- `ai/rules/curaos_image_build_rule.md:22` says "Multi-arch builds (amd64 + arm64) | both".
- `ai/rules/curaos_image_build_rule.md:81-87` shows mandatory `--platform linux/amd64,linux/arm64`.

Why it breaks contract:
M8-S1 pins layout consumed by M8-S4 image build/signing and M8-S6 install/rollback. amd64-only excludes arm64 and makes 5.77 GB / 7.37 GB budget non-binding for release bytes.

Required fix:
Change architecture to `multi`, account for linux/amd64 + linux/arm64 in budget, document OCI index vs per-platform digests, and recompute sizes.

### P0-2 — CNPG layer violates locked PostgreSQL stack: no Citus, stale MinIO target, no SeaweedFS path

Evidence:
- `ops/zarf/zarf.yaml:100-113` defines CNPG + `postgresql:16.4` + pgBouncer only.
- `ops/zarf/zarf.yaml:55-57` defines `BACKUP_S3_ENDPOINT` as "MinIO endpoint" with default `minio.curaos-ops.svc.cluster.local`.
- `ai/rules/curaos_postgres_rule.md:18` says "**CloudNativePG (CNPG) + Citus extension are the only PostgreSQL stack for CuraOS K8s clusters**".
- `ai/rules/curaos_postgres_rule.md:23` requires "**Citus distributed PG** — coordinator + 3-5 worker nodes".
- `ai/rules/curaos_postgres_rule.md:27` requires "**SeaweedFS S3** ... replaces MinIO per DA13 Q6".
- `ai/rules/curaos_postgres_rule.md:77-88` explains MinIO rejected for air-gap bundle risk.

Why it breaks contract:
PR binds downstream M8 stories to pre-DA13 PostgreSQL shape. M8-S3/M8-S4 would generate/sign wrong DB operand; M8-S6 PITR would target replaced storage.

Required fix:
Include Citus-compatible Postgres operand and required Citus/bootstrap artifacts, model coordinator/worker topology, keep pgBouncer via CNPG `Pooler`, replace MinIO with SeaweedFS S3, or explicitly reconcile single-tenant exception with rule.

### P0-3 — K3s import does not encode required Cilium-safe install flags

Evidence:
- `ops/zarf/zarf.yaml:68-75` imports `oci://ghcr.io/zarf-dev/packages/init:v0.76.0` component `k3s`.
- No `--flannel-backend=none`, `--disable-network-policy`, or `--disable-kube-proxy` appears in PR #88 changed files.
- `ai/rules/curaos_cni_rule.md:28` bans Flannel and kube-proxy iptables mode.
- `ai/rules/curaos_cni_rule.md:35-44` requires K3s with `--flannel-backend=none`, `--disable-network-policy`, `--disable-kube-proxy`.
- `ai/rules/curaos_cni_rule.md:46-58` then installs Cilium with `kubeProxyReplacement=true`.

Why it breaks contract:
Stock K3s then Cilium later is not equivalent to Cilium-safe bootstrap. Default Flannel/kube-proxy can become load-bearing before Cilium lands, invalidating zero-egress/network-policy baseline.

Required fix:
Pass or wrap exact K3s flags from CNI rule, ensure Cilium values set `kubeProxyReplacement=true` and API host/port. If upstream init package cannot expose flags, vendor CuraOS-specific K3s init package.

- Multiple image refs use `@sha256:<digest>` at `ops/zarf/zarf.yaml:92-94`, `110-112`, `128-129`, `145-152`, `173-174`, `204-209`, `228-237`.
- `ai/rules/curaos_version_pinning_rule.md:14` requires cosign-signed OCI image digests as canonical lock.
- `ai/rules/curaos_version_pinning_rule.md:35` says digest pins are immutable and cosign-verifiable.
- `ai/rules/curaos_version_pinning_rule.md:158-165` requires `@sha256:<64-hex-digest>`.
- `ai/curaos/docs/adr/0164-zarf-bundle-layout.md:111-112` says refs carry placeholders pending M8-S4.

Impact:
Acceptable only as non-releasable spike. Current wording blurs placeholder shape with digest-pinned manifest.

Required fix:
Mark manifest non-packageable until placeholders replaced, add CI grep failing on `<digest>`, and say refs must be replaced by 64-hex digests before package create/sign/deploy.
