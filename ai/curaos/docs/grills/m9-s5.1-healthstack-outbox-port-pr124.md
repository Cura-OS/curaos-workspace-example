## Resolution detail

1. **Atomicity (CRITICAL).** The chain-head CAS and the outbox enqueue MUST share ONE commit boundary. The fix mirrors the canonical identity-core `audit/` stack (the more-evolved sibling of the `db/` stack): the chain-head store's `get` + `compareAndSet` accept an optional tx-bound Drizzle executor (`db?`), and `AuditPublisherService.publish(input, tx?)` threads `tx?.db` into BOTH the head read/CAS and the outbox `tx.enqueue`. Order inside the tx: read head → compute hash → CAS (throw + rollback on lose, so NO orphan row) → enqueue (durable-before-ack). In-memory/file backends ignore `db` (no real Postgres tx) — for those the publish is a single in-process call so neither a hole nor an orphan can be observed across a crash boundary. The Postgres path closes the window completely.

2 + 3. **Producer.send relocation (MAJOR/OK).** The relay (`AuditOutboxRelayService`) becomes the sole Kafka delivery path; `publish()` no longer calls `producer.send()`. A Kafka outage is invisible to `publish()` — the row is committed durable, and the relay retries on the next tick (markFailed → still pending w/ backoff until maxRetries). The old `audit-chain-kafka-failure.test.ts` (which asserted `publish()` throws on Kafka outage) and `audit-chain-concurrent.test.ts` (per-resource lock) are deleted; new outbox-pattern integration tests assert: relay drains pending rows, a relay-publish failure keeps the row pending (durability), pending rows replay on boot (restart survival), and concurrent same-resource publishes produce a contiguous chain with exactly one row per step + no orphan from a losing CAS.

4. **`audit_leg` retained (MAJOR).** Keep `audit_leg text NOT NULL` in the migration + `auditLeg` in the store types to preserve the canonical codegen shape. Healthstack is single-leg, so every row is written with the constant leg `'healthstack'` (identity specializes to `'m3'|'diamond'`).

5. **Generator-evolution (MAJOR).** Hand-port is correct: `drizzle.config.ts` `schemaFilter: ['healthstack']` excludes this service from regen, and the codegen template emits `{{snakeCase name}}_core`. The hand-port overrides the schema to `healthstack` + single-leg. Codegen fold-back noted: the canonical `service-core` chain-head template lacks the tx-bound `db?` CAS parameter that the outbox tx-threading requires — folding the tx-bound CAS + the single-leg-default `audit_leg` convention back into `tools/codegen/templates/service-core/src/audit/audit-chain-head.store.ts.hbs` is a generator concern tracked separately (does not block this story; healthstack inherits the already-correct identity-core `audit/` shape by hand).

### Atomicity end-to-end: OPEN (P1 — pre-existing, not introduced by 149f110)

Call stack: controller awaits service (`src/patients/healthstack-patients.controller.ts:97`, `:106`, `:124`, `:139`) → service repository insert/update occurs at `src/patients/healthstack-patients.service.ts:147`/`:276` → `publishAudit()` called after mutation at `:175`/`:179`/`:300`/`:304`. The repository mutates state immediately in its own transaction; `audit.publish()` is invoked in a separate call outside any shared DB transaction (`src/audit/audit-publisher.service.ts:169`, `:174`; `src/patients/healthstack-patients.service.ts:333`).

Impact: if the durable audit enqueue throws after the repository commit, the PHI mutation is persisted but no durable audit row exists. The caller receives an error (fail-closed — no silent success ACK), but the state is already written. A retry will succeed and produce an audit row, leaving a window where the committed PHI state has no corresponding audit entry. This is the classic write-then-audit gap for HIPAA durability.

Correct fix: wrap repository mutation + `audit.publish()` in one DB transaction, or implement saga-style compensating rollback on audit failure.

This gap existed before 149f110. The original grill did not flag it as a P0 (it was subsumed in the test quality finding). It is raised here as a P1 blocker for a HIPAA audit path.

Defer all external emission (`HealthstackPatientEventProducer.send`) to a post-commit relay from the outbox before merging.
