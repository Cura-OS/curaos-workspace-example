### Check 8 — Signature co-location / Zarf registry rewrite — NOW PASS

**Architectural research** (Zarf upstream wiki + source via DeepWiki `zarf-dev/zarf`):
- At air-gap deploy time, `zarf-agent` mutating webhook rewrites every PodSpec image ref from `ghcr.io/<path>:tag@sha256:<digest>` to `zarf-docker-registry.zarf.svc.cluster.local:5000/ghcr.io/<path>:tag@sha256:<digest>` — FQDN replaces registry hostname, original image path preserved verbatim, digest preserved. NodePort form `127.0.0.1:31999` is operator-side external only.
- Zarf v0.76 does NOT auto-bundle cosign `.sig` OCI artifacts — `zarf package create` only signs `zarf.yaml` itself. The `.sig` artifact distribution into the in-cluster mirror is M8 P2 followup (filed in ADR-0211 §8).
- Digest preservation guarantees cosign signature reachability when `.sig` IS mirrored (signature is content-addressed by digest).
