## P1

- **P1, `ops/zarf/charts/curaos-umbrella/values.yaml:11`: partial digest pinning ships a false immutable umbrella.**
  The umbrella emits mutable `global.image.tag: "0.1.0-spike"` at `values.yaml:11-13`, then only 32 per-service digest blocks from `values.yaml:14-109`, while every service remains enabled from `values.yaml:113-291`. The checked chart has 89 service dependencies from `Chart.yaml:17-373`, including unpinned enabled examples `audit-service` and `automation-core-service` at `Chart.yaml:26-33` plus `values.yaml:118-121`.
  Exact break: 57 enabled services silently render as `repo:0.1.0-spike` (mutable tag), giving a false sense of digest immutability across the umbrella.
  Minimal fix: make `gen:umbrella` fail unless every enabled service has a backfilled digest, OR require an explicit noisy partial-pin mode that is not used by release/Zarf paths.

## P2

- **P2, `tools/codegen/templates/service-core/chart/templates/_helpers.tpl:71`: `global.image.digest` is a live multi-service footgun (not auto-emitted).**
  The helper takes `$global.digest` before `.Values.image.digest` at line 71, then renders `repo@<digest>` at lines 73-74. The emitter emits `global.image.tag` at `umbrella-emit.ts:316-318` and per-service `<slug>.image.digest` at `umbrella-emit.ts:320-326`, and NEVER `global.image.digest` (confirmed dead for the catastrophic auto-emit case).
  Exact break: a manual `--set global.image.digest=sha256:...` overrides every per-service digest with the SAME digest, pinning all 89 services to one image.
  Minimal fix: remove `global.image.digest` support from generated multi-service charts, or `fail` when it is set in umbrella installs.

## P3

- **P3, `tools/codegen/templates/service-core/chart/templates/_helpers.tpl:73`: malformed non-empty digest not validated before forming `repo@...` (empty strings are safe).**
  Defaults set `digest: ""` at each `values.yaml.hbs:22`; empty `$digest` skips to `repo:<tag>` at helper lines 73-76. Any non-empty malformed digest passes the truthy branch and renders `repo@<malformed>`.
  Minimal fix: validate non-empty digests with `^sha256:[0-9a-f]{64}$` and `fail` on invalid input.

4. Empty/malformed digest: empty = NON-ISSUE; malformed non-empty = REAL, P3.
5. Per-service digest vs global.tag interaction: NON-ISSUE; digest wins at helper lines 71-74.
