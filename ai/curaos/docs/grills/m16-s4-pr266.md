## Findings + auto-applied decisions

### D1 (decision, auto-applied): component 0 `zarf-registry-init` import is wrong + mislabeled
- **Where:** `ops/zarf/zarf.yaml:97-103`.
- **What:** imports `name: zarf-injector` from the upstream init OCI package. Verified
  against upstream v0.76.0 sources: the `zarf-injector` component is `only.cluster
  .architecture: amd64` and is the EPHEMERAL Rust bootstrap binary ("Removed and destroyed
  after the Zarf Registry is self-hosting the registry image"). It is NOT the registry
  mirror and NOT the agent webhook. The component description claims "registry mirror +
  agent webhook" which is false for that import. A single Zarf `import:` block imports
  exactly ONE component, so no single import can compose registry-mirror + agent-webhook.
  The bundle is `architecture: multi` (amd64 + arm64), so importing the amd64-only injector
  is also arch-invalid.
- **Recommendation (Codex, auto-applied):** Remove the upstream init OCI import from
  component 0. Keep the component NAME `zarf-registry-init` so the 13-component
  `zarf-deploy-order-check.sh` guard + ADR-0164 count stay green (no T3 ADR/guard change).
  Make it a CuraOS-owned actions-only PREFLIGHT component that asserts the cluster was
  Zarf-initialized (the standard `zarf init` operator step bootstraps registry + agent +
  injector for the operator's arch). Document the real bootstrap as the README operator
  flow: `zarf init oci://ghcr.io/zarf-dev/packages/init:v0.76.0` (or an external-registry
  init) BEFORE `zarf package deploy` of the CuraOS application package. Correct the
  description + header comments to match reality. This matches the standard Zarf model
  (`zarf init` is a separate operator step from `zarf package deploy`).
- **Why not import `zarf-registry`/`zarf-agent` instead:** `zarf-registry` is closer to
  "mirror" but still omits the agent webhook and depends on the seed/injector flow;
  `zarf-agent` is webhook-only. Neither single import satisfies the stated purpose, and a
  full multi-component init flow would change the component count (T3 ADR/guard work, out of
  scope for this story).
