# CuraOS Doc Graph Rule

Canonical rule: CuraOS docs are an LLM-maintained wiki, not a pile of standalone Markdown files.

## Rule

Every Markdown document is a graph node. Every new or changed document must keep relationship edges current in `ai/curaos/docs/DOC-GRAPH.md`.

The graph must preserve a directed relationship path from workspace `AGENTS.md` to every Markdown node. New docs that are not reachable from `AGENTS.md` fail verification.

The model follows the Karpathy LLM Wiki pattern:
- Raw/historical sources stay available.
- Current wiki docs synthesize the latest state.
- Pages are interlinked and navigable by agents.
- Lint catches broken links, orphan pages, and stale graph output.

## Required Behavior

When adding, moving, renaming, or materially editing any `.md` file:

1. Add or keep meaningful internal relationships:
   - Markdown links for exact file references.
   - `[[curaos-...-rule]]` wikilinks for canonical rule references.
   - ADR links for decisions.
   - `AGENTS.md` / `CONTEXT.md` / `Requirements.md` sibling links for module docs.
2. Run:

```bash
bun scripts/check-doc-graph.js --write
bash scripts/check-docs.sh
```

3. Commit the updated `ai/curaos/docs/DOC-GRAPH.md` with the doc change.

CI runs `.github/workflows/docs.yml` on Markdown and verifier changes. The workflow fails when `DOC-GRAPH.md` is stale, links break, or active docs reintroduce superseded stack terms.

## Relationship Types

The verifier emits typed edges:

| Edge | Meaning |
|---|---|
| `governs` | Workspace `AGENTS.md` reaches a Markdown node |
| `governed_by` | Workspace `AGENTS.md` applies |
| `repo_contract` | Repo-level `ai/curaos/AGENTS.md` applies |
| `module_contract` | Sibling/module `AGENTS.md` applies |
| `contextualized_by` | `CONTEXT.md` gives current integration map |
| `specified_by` | `Requirements.md` gives module spec |
| `indexes_rule` / `indexed_by` | `ai/rules/README.md` indexes rule files |
| `indexes_adr` / `status_in` | `RESOLUTION-MAP.md` indexes ADR status |
| `references` | Explicit Markdown link |
| `wikilink` | `[[...]]` rule/ADR style reference |

## Failure Modes Prevented

- Orphan docs that agents never discover.
- Stale ADRs overriding newer rules.
- New docs with no inbound/outbound relationship.
- Broken links after renames.
- Research notes being treated as current decisions.
