ADR-0100 status (2026-05-24):

**RECOMMENDATION NOT YET FINAL.** User clarification: Go + Rust surfaced as candidates from comfort-zone + priorities filter, but not decided. More exploration needed before commit.

**Hot-reload + customization architecture (user-directed pattern):**

CuraOS adopts a **stable-core + plugin/sidecar** model:
- **Stable core (host service)** = compiled (whichever runtime wins) - rarely changes, security patches + dep upgrades only, Helm rolling deploy
- **Plugin/sidecar layer** = managed + generated by CuraOS platform - carries tenant logic + business customization + extension code
- **Hot-reload combo (all three layers):**
  1. **BPMN flows** (Flowable / Camunda / Temporal / other) - hot-reload process definitions
  2. **DMN decisions** - hot-reload decision rules
  3. **WASM modules + embedded scripts** - hot-reload code-level customization
  4. **Helm rolling deploy** - for stable-core upgrades

**Implication:** the platform must generate both:
- Plugin/SPI scaffold (per chosen runtime)
- Sidecar scaffold (likely Unix-domain-socket or gRPC localhost)
- Hot-loadable customization modules (WASM/scripts)
- BPMN/DMN definitions wired to the host

**Codegen requirement reframe:** the generic codegen platform (per user spec earlier - CRUD + forward/backward engineering + interceptor + BPM tie-in) must emit:
1. Stable-core service code (per chosen runtime, OSS scaffolds reused)
2. Plugin/sidecar scaffold + SPI bindings
3. BPMN/DMN templates for customization points
4. WASM/script harness for custom code

**Decision sequencing now:**
1. Re-examine candidate runtimes with stable-core + plugin/sidecar lens. Some runtimes do this better than others (Go plugins vs Rust dylib vs JVM hot-swap vs WASM-native).
2. Re-examine codegen ecosystem for whichever runtime - must support the multi-tier emit.
3. Re-examine hot-reload mechanisms - WASM hosts (Wasmtime/Wazero/WasmEdge) work with any host language; Lua/Starlark embed in Go natively; Rhai embeds in Rust.

**Open questions for next interview round:**
- Sidecar mesh: how do sidecars discover + communicate with stable core? (Spring Sidecar pattern? Dapr? Istio? Linkerd? Custom?)
- Plugin lifecycle: version compatibility, dependency resolution, security sandbox for tenant-supplied plugins
- Generate-time vs runtime: are plugins generated by platform at tenant onboarding (compile-once-then-load) or runtime-interpreted (WASM/script)?
- Cross-runtime customization: does same WASM module work across Go + Rust + JVM hosts? (Yes if pure WASM, but performance varies)
