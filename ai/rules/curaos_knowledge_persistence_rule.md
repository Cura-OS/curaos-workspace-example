## The rule

**Six locked components:**

1. **6-layer knowledge architecture** - Session / Module / Decision / Structural / Temporal / Cold
2. **MADR 4.0.0 ADR format** w/ YAML frontmatter (queryable via `yq`)
3. **HANDOVER.md overwrite-per-session** - point-in-time snapshot; git history IS the log
4. **Per-module CONTEXT.md <500 lines** - codified context tier hot/warm/cold per arXiv:2602.20478
5. **codegraph MCP only** for structural queries (NO memory MCPs per DA3)
6. **GLOSSARY.md + runbooks + postmortem → CONTEXT.md** discipline

## Live-state precedence (amendment 2026-06-10, remediation RP-24)

For LIVE EXECUTION STATE (active milestone, in-flight issues, blockers, claim/PR status), HANDOVER + tracker WIN over auto-memory and over CONTEXT.md-class phase text:

1. **Tracker** (GitHub Issues + Project board) - source of truth.
2. **`ai/curaos/docs/HANDOVER.md` + `ai/curaos/docs/ISSUE-ROADMAP.md`** - tracker mirrors, regenerated by `scripts/session-closeout` at every close.
3. **Everything else** (auto-memory MEMORY.md + session topic files, CONTEXT.md phase lines, README prose) is HISTORY; when it disagrees with 1-2 it is stale by definition and must not drive milestone-state decisions.

Incident basis: session-28 acted on stale auto-memory for milestone state, and workspace CONTEXT.md still claimed a long-closed milestone was in flight at deep-review time (RISK-REGISTER RR-04). Guard: static docs carry POINTERS, not hardcoded milestone numbers; the standing pointer block lives at `ai/curaos/docs/README.md` ("Current state") and `scripts/session-closeout` fails when that block is missing.

## Banned

- Memory MCPs (Anthropic KG, Mem0, Letta, Zep, Cipher) - per [[curaos-mcp-stack-rule]] DA3 (file-based + codegraph sufficient)
- HANDOVER.md append-only (use overwrite-per-session; git history IS the log)
- CONTEXT.md >500 lines (silent truncation; effective context window burn)
- Auto-generated CONTEXT.md (LLM-emitted content gives negative returns per [[curaos-agents-md-schema-rule]] empirical -4%)
- ADR by title reference in code comments (use number per [[curaos-repo-conventions-rule]])
- Architectural decisions in GitHub Wiki (use in-repo ADRs)
- Postmortem-only learnings (extract to CONTEXT.md "Known failure modes" for agent-facing summary)
- Recording the same lesson a second time without adding/updating its `ai/curaos/docs/RISK-REGISTER.md` row (prose-only lessons recur; register rows get guards)
- Glossary terms invented per-conversation (canonical definitions in GLOSSARY.md only)
- AGENTS.md inside submodule repos (lives in ai/curaos/ mirror per [[curaos-ai-mirror-rule]])
- Onboarding doc order skipped (1→2→3→...→7→code order matters; agents miss context if skipped)
- ADRs deleted when superseded (mark `status: superseded` + `superseded-by: ADR-XXXX`; reasoning trail valuable)

## Why

| Constraint | Empirical / mechanical backing |
|---|---|
| 6-layer architecture | Each layer answers different query class; agents query layer matching question type (session/module/decision/structural/temporal/cold); avoids re-reading wrong artifact |
| MADR 4.0.0 | YAML frontmatter enables `yq` queries across decision store ("all accepted ADRs about databases"); consistent section naming; cross-referencing support |
| HANDOVER.md overwrite-per-session | Point-in-time snapshot - git history IS the log; appending grows stale (stale info accumulates); current state always one file read |
| CONTEXT.md <500 lines | Agents read in full; >500 lines = silent truncation OR full context window burn; tier-3 codified context architecture (arXiv:2602.20478) keeps hot tier focused |
| codegraph MCP only | 57% fewer tokens + 71% fewer tool calls + 46% faster vs traditional agent file-reading (CodeGraph benchmarks); generic memory MCPs banned per [[curaos-mcp-stack-rule]] EXCEPT the self-hosted mem0 backend adopted 2026-06-21 per [[curaos-mem0-memory-backend-rule]] (mirror only; ai/rules/ + file-memories stay canonical; no PHI in mem0) |
| AGENTS.md adoption 29% runtime reduction | Empirical: AGENTS.md presence reduces median agent runtime 29% + token consumption 17% (per [[curaos-agents-md-schema-rule]]) |
| Conventional Commits + git pickaxe | `git log -S "<pattern>" --oneline` finds when X introduced w/o reading every ADR; temporal query layer |

## Layer 1 - Session (volatile; survives /compact)

### Files

```
~/.claude/CLAUDE.md                     # user-global behavioral rules (always loaded)
<workspace>/CLAUDE.md                   # @AGENTS.md import + Claude-specific addenda
<workspace>/AGENTS.md                   # workspace contract + rules index (per [[curaos-agents-md-schema-rule]])
~/.claude/projects/<repo>/memory/MEMORY.md  # auto memory index (first 200 lines / 25KB reloaded post-compact)
~/.claude/projects/<repo>/memory/<topic>.md # auto memory topic files (loaded on demand)
<workspace>/HANDOVER.md                 # session-end snapshot (overwrite-per-session)
```

### Discipline

- CLAUDE.md <200 lines (Anthropic limit; full load every session)
- `.claude/rules/*.md` path-scoped (YAML `paths:` field); load on file match
- Auto memory `~/.claude/projects/<git-repo>/memory/MEMORY.md` - Claude writes from corrections (per [[curaos-memory-agents-sync-rule]])
- `/compact` proactive at 60% (per [[curaos-context-engineering-rule]])
- Root CLAUDE.md auto-re-injects post-compact; nested subdir CLAUDE.md reloads on file read there

## Layer 2 - Module (per-module agent contract + context)

### Files (per [[curaos-agents-md-schema-rule]] DA2)

```
ai/curaos/<module-path>/
├── AGENTS.md      # YAML frontmatter (extended CuraOS schema) + ASDLC strict body - <150 lines
├── CONTEXT.md     # Non-obvious knowledge, failure modes, decision trees, patterns - <500 lines
└── Requirements.md # Charter, spec, done criteria - no size cap
```

### CONTEXT.md content rules

**Include:**
- Project-specific conventions + decisions (not general programming knowledge)
- Actual failure modes from debugging sessions
- Decision trees (delivery mode, error handling)
- Concrete code examples w/ real file paths + identifiers
- Known gotchas + footguns
- "If explained twice, write down" - repeated explanations signal missing context

**Exclude:**
- General programming knowledge agents already possess
- Architecture summaries belonging in ADRs
- Outdated patterns (stale specs actively mislead - worse than no spec)
- Extensive explanations suited to external docs

### Codified context tier architecture (arXiv:2602.20478)

```
Tier 1 (Hot): ~660-line constitution always in context
              - AGENTS.md frontmatter + Toolchain Registry + Judgment Boundaries
Tier 2 (Warm): 19+ domain expert specs (115-1,233 lines)
              - CONTEXT.md per module + topic-specific rules
Tier 3 (Cold): 34+ on-demand docs accessed via MCP/RAG
              - postmortems, research/, deprecated specs
```

## Layer 4 - Structural (codegraph MCP only)

Per [[curaos-mcp-stack-rule]] DA3 must-have MCP: codegraph for structural queries.

### Use cases

- "What calls function X?" → `codegraph_callers`
- "What does Y call?" → `codegraph_callees`
- "What would break if I changed Z?" → `codegraph_impact`
- "Where is X defined?" → `codegraph_search`
- "Show me Y's signature/source/docstring" → `codegraph_node`
- "Give me focused context for an area" → `codegraph_context`
- "See several related symbols' source" → `codegraph_explore`

### Benchmarks (vs traditional agent file-reading)

- 35% cheaper cost
- 57% fewer tokens
- 71% fewer tool calls
- 46% faster responses

### Discipline

- Index lag: file watcher debounces ~500ms - no re-query immediately after editing
- Staleness signal: `codegraph_status` reports "not initialized" → run `codegraph init -i`
- Rebuild in CI on merge to main
- Local-only (no external traffic; PHI-safe)

### Banned (per [[curaos-mcp-stack-rule]] DA3)

- Anthropic KG Memory MCP, Mem0, Letta, Zep, Cipher - redundant w/ file-based persistence
- Sourcegraph MCP - enterprise license + cloud dep
- tree-sitter MCP - codegraph subsumes
- ast-grep MCP - use `sg` CLI (per [[curaos-mcp-stack-rule]] CLI-first)

## Layer 5 - Temporal (git log + Conventional Commits + git pickaxe)

### Conventional Commits as searchable history

Per [[curaos-repo-conventions-rule]]: `type(scope): subject`

```bash
git log --grep="feat(identity)" --oneline      # all identity feature additions
git log --grep="fix(auth)" --since="2025-01-01" # auth fixes since date
git log -S "pgBouncer" --oneline               # when pgBouncer reference appeared (pickaxe)
git log -G "RLS" --oneline                     # commits changing RLS-related code
git log -L :functionName:path/to/file          # history of a specific function
```

### Code archaeology workflow

1. `rg <pattern>` → find where appears (or `codegraph_search`)
2. `git log -S "<pattern>" --oneline` → find when introduced
3. `git show <hash>` → full context of introduction
4. `git log --follow -p -- <file>` → full file history including renames

### git-cliff CHANGELOG

Per [[curaos-repo-conventions-rule]]: agent-readable; reproducible; stays current automatically.

### Pattern: agent hits unusual code

`git log -S "<pattern>" --oneline` + `git show <hash>` reconstructs decision context w/o separate ADR. Combine w/ Layer 3 (DECISION-LOG.md) for full picture.

### RISK-REGISTER.md (recurring-failure register, BINDING)

[`ai/curaos/docs/RISK-REGISTER.md`](../curaos/docs/RISK-REGISTER.md) holds one row per recurring failure pattern: {pattern, incidents, guard (script/gate path or NONE), last recurrence}.

- **Binding:** any lesson recorded twice (memory, HANDOVER, postmortem, session retro) MUST get a register row in the same session that records the second occurrence.
- Rows with guard = NONE older than 3 sessions auto-convert to staged foresight issues per [[curaos-foresight-rule]] (LESSON-mining sweep, remediation item RP-45; manual conversion applies until that sweep lands).
- Evidence basis: patterns that received converger scripts stopped recurring; prose-only lessons all recurred (see register seed rows RR-13 through RR-17).

## Persistent memory vs lookup

**Persistent (CLAUDE.md / CONTEXT.md):**
- Project-specific commands differing from conventions
- Non-standard tooling (Bun not npm; Drizzle not Prisma)
- Known failure modes + footguns
- PHI/PII boundary rules (cannot be re-derived safely)
- Module dependency direction constraints

**Lookup at task time:**
- API signatures → codegraph or TypeDoc output
- Architectural decisions → RESOLUTION-MAP.md (current live index) → specific ADR (DECISION-LOG.md planned but not yet created)
- Historical reasoning → git log pickaxe search
- Postmortem details → ai/curaos/docs/postmortems/
