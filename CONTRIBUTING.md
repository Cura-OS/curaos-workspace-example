# Contributing

Thanks for improving the CuraOS workspace example.

## Scope

Accepted contributions should stay limited to sanitized workspace patterns, documentation, templates, and checks. Do not add private CuraOS product code, internal roadmap details, credentials, customer data, or infrastructure secrets.

## Standards

- Keep examples generic and reusable.
- Preserve the doc graph.
- Avoid em dash and en dash characters.
- Run available checks before opening a pull request.
